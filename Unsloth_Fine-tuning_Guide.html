什么是模型微调？

微调大语言模型是一种定制化 AI 行为、增强知识和优化特定领域性能的强大技术。通过微调，您可以将通用模型转变为专门针对特定任务的专家系统，显著提升其在目标领域的表现。

您可以将微调后的模型视为一个专门设计的智能代理，能够更有效、更精确地执行特定任务。微调不仅可以复制 RAG（检索增强生成）的所有功能，而且能够提供 RAG 无法实现的能力。

微调的主要应用场景

  • 训练模型预测新闻标题对公司的正面或负面影响
  • 利用历史客户互动数据，提供更准确和个性化的响应
  • 基于法律文本微调模型，用于合同分析、案例法研究和合规检查

微调全流程指南

1

选择合适的模型和方法

对于初学者，建议从小型指令模型如 Llama 3.1 (8B) 开始实验。您还需要在 QLoRA 和 LoRA 训练方法之间做出选择。

QLoRA

使用量化技术减少内存需求，适合在有限资源环境下训练大型模型。

LoRA

低秩适应方法，通过训练少量参数实现高效微调，保持模型性能的同时减少计算需求。

2

准备数据集

数据集的质量和数量将直接影响微调结果，因此这一步至关重要。

  • 通常需要创建包含问题和答案两列的数据集。数据质量和数量将直接反映在微调结果中。
  • 可以使用 ChatGPT 或本地 LLM 合成数据并将数据集结构化为问答对。
  • 微调可以永久性地整合现有文档库并持续扩展知识库，但仅仅堆砌数据效果不佳。为获得最佳结果，应精心策划结构良好的数据集，理想情况下为问答对形式。
3

理解模型参数

掌握关键参数对于避免过拟合和欠拟合至关重要。

参数 描述 建议值
学习率 控制模型学习速度 2e-4 至 5e-5
批次大小 每次更新处理的样本数 1 至 8
训练轮次 完整数据集的训练次数 3 至 5
LoRA 秩 低秩矩阵的维度 8 至 64

避免过拟合与欠拟合

过拟合（太具体）

模型过度记忆训练数据，无法泛化。解决方案：

  • • 减少训练轮次
  • • 增加数据多样性
  • • 使用正则化技术
欠拟合（太泛泛）

模型未能从训练数据中学习，回答类似基础模型。解决方案：

  • • 增加学习率
  • • 延长训练时间
  • • 使用更相关的领域数据集
4

训练与评估

训练过程中，您将看到损失值不断下降，这表明模型正在学习。评估阶段至关重要，可以验证模型是否达到预期效果。

评估方法

  • 人工评估：检查模型回答质量
  • 自动评估：使用指标如准确率、F1分数
  • 对比评估：与基础模型比较性能

评估指标

  • 回答准确性
  • 回答相关性
  • 领域知识应用
  • 生成文本质量
5

保存与部署模型

完成微调后，您可以将模型保存为小型 LoRA 适配器文件（约 100MB），或上传到 Hugging Face。


# 保存为 LoRA 适配器
model.save_pretrained("path/to/save/model")

# 上传到 Hugging Face
model.push_to_hub("your-username/model-name")

常见问题解答

微调可以复制 RAG（检索增强生成）的所有功能，但 RAG 无法实现微调的所有能力。微调通过修改模型权重永久性地整合知识，而 RAG 在推理时动态检索信息。微调后的模型可以更深入理解领域知识，表现出更一致的行为模式。

对于初学者，建议从小型指令模型如 Llama 3.1 (8B) 开始实验。这类模型已经具备良好的基础能力，同时对计算资源要求较低。随着经验积累，可以尝试更大的模型以获得更好的性能。

数据需求取决于任务复杂度和特定领域。一般而言，几百到几千个高质量样本就可以取得不错的效果。关键是数据质量而非数量，精心策划的小型数据集通常比大量低质量数据更有效。对于特定领域任务，甚至100-500个精心设计的样本就能显著提升性能。

避免过拟合的策略包括：使用更多样化的训练数据、减少训练轮次、应用早停技术、使用正则化方法（如权重衰减）、适当降低学习率，以及使用验证集监控性能。定期评估模型在未见过的数据上的表现也很重要。

Tim Dettmers 等
